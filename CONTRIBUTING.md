# Contribution Guidelines

## Architecture Freeze
- The architecture defined in docs/MASTER_ARCHITECTURE.md is final
- No new top-level folders are allowed
- No technology stack changes without team approval

## Branching Rules
- The main branch is protected
- All development must be done in feature branches
- Pull requests are required for merging

## Documentation Rules
- Each module must maintain its README.md
- Any architectural change must update MASTER_ARCHITECTURE.md

## Code Responsibility
- Members must work only within their assigned modules
- Integration is handled through reviewed pull requests
